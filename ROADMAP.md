# 🏁 NFStest: План разработки (Roadmap)

Проект аркадной гонки в стиле NFS Most Wanted (2005). 
**Стек:** Unity 6000.3.10f1, C#, Git (GitHub).

---

## 🛠 ЭТАП 0: Инфраструктура и Архитектура
*Эти задачи закладывают фундамент, чтобы не переписывать код потом.*

- [ ] **Настройка Git LFS**: Конфигурация для `.fbx`, `.png`, `.mat` (чтобы репо не тормозил).
- [ ] **Архитектура данных**: Реализация `ScriptableObjects` для характеристик машин (Speed, Torque, Handling).
- [ ] **Система ввода (Input System)**: Настройка нового Unity Input System для поддержки клавиатуры и геймпада.
- [ ] **Folder Structure**: Приведение папок к стандарту (`_Project/Scripts`, `_Project/Prefabs`, `_Project/Models`).

---

## 🚗 ЭТАП 1: Физика и Игрок (Player Core)
- [ ] **Прототип контроллера**: Реализация движения через `WheelColliders`.
- [ ] **Переключение передач**: Логика автоматической/ручной КПП.
- [ ] **Camera System**: Настройка `Cinemachine` (динамическое отдаление при разгоне, эффект тряски).
- [ ] **Визуализация**: Подключение моделей машин, настройка материалов (кузов, фары, стекла).

---

## 🛣 ЭТАП 2: Окружение и Трассы (World Design)
- [ ] **Greyboxing трассы**: Сборка первого трека из примитивов (ProBuilder) для теста скорости.
- [ ] **Spline System**: Настройка путей для дорожных ограждений и разметки.
- [ ] **Environment**: Замена кубов на меши дорог, добавление простых зданий и освещения.
- [ ] **Пост-процессинг**: Настройка Bloom и Color Grading для атмосферы "золотого часа" из MW.

---

## 🤖 ЭТАП 3: AI и Трафик (NPC)
- [ ] **Система путей (Waypoints)**: Граф дорог, по которому движется городской трафик.
- [ ] **Simple Traffic AI**: Логика NPC (движение по линии, остановка перед игроком).
- [ ] **Police AI (FSM)**: Базовая машина копов с состояниями: `Patrol` -> `Chase` -> `Ram`.

---

## 🎮 ЭТАП 4: Игровой цикл и UI
- [ ] **HUD**: Спидометр (UI Toolkit или Canvas), мини-карта.
- [ ] **Система чекпоинтов**: Логика кругов и фиксация времени.
- [ ] **Game Manager**: Состояния игры (Меню -> Гонка -> Финиш).

---

## 🤝 ПРАВИЛА РАБОТЫ В КОМАНДЕ (Git Workflow)
1. **Никогда не мержить `.unity` файлы (сцены)** напрямую. Работаем через **Prefabs**.
2. Изменил префаб машины — сделал коммит.
3. Перед началом работы всегда делаем `git pull`.
4. Для крупных фич (например, копы) создаем отдельную ветку: `git checkout -b feature/police-ai`.
